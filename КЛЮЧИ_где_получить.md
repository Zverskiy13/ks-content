# Ключи и переменные контент-платформы — где взять

Всё вписывается в **Variables** сервиса контент-платформы (`ks-content`) в Railway. Имена печатать вручную, без пробелов.

---

## 1. `DATABASE_URL` (обязательно)
Публичная строка того же Postgres, что у приложения.
- Railway → проект **keen-learning** → сервис **Postgres** → вкладка **Variables** → скопируй значение **`DATABASE_PUBLIC_URL`** (хост НЕ `postgres.railway.internal`, а вида `xxxxx.proxy.rlwy.net:12345`).
- Вставь как `DATABASE_URL` в сервис контент-платформы.

## 2. `SECRET_KEY` (обязательно)
Любая длинная случайная строка (подпись входа). Можешь взять эту:
```
WUSsoXsJdp58pYR1vpon9fCcwvlGp-SkJks8VYU_LZMWNeJBhxx0OCHN3-Ea7XLP
```

## 3. `CONTENT_OWNER_EMAIL` и `CONTENT_OWNER_PASSWORD` (обязательно)
Придумываешь сам — это твой первый вход владельцем. E-mail (напр. `ivkuzin11@gmail.com`) + пароль (запиши).

## 4. `ANTHROPIC_API_KEY` (обязательно — весь ИИ: аналитик, сценарист, советы)
- Открой **https://console.anthropic.com** → войди → слева **API Keys** (или https://console.anthropic.com/settings/keys) → **Create Key** → скопируй.
- Это платный ключ (оплата картой). Тот же ключ, что у приложения — можно переиспользовать.

## 5. `VK_SERVICE_TOKEN` (для авто-сбора роликов VK в «Разведке»)
- **https://vk.com/apps?act=manage** → **Создать приложение** → тип **Standalone**.
- В приложении: **Настройки** → скопируй **«Сервисный ключ доступа»**.
- Только чтение публичных сообществ — этого достаточно для разведки.

## 6. Токен сообщества VK — для АВТОПОСТИНГА (вводится в самой платформе, не в Railway)
- Зайди в своё сообщество VK → **Управление** → **Настройки** → **Работа с API** → **Ключи доступа** → **Создать ключ**.
- Отметь права: **Управление сообществом**, **Стена**, **Фотографии**, **Видеозаписи**.
- Этот ключ + числовой **ID группы** вводишь в платформе: вкладка «Контент-план» → «Подключение соцсетей».
- ⚠️ Если загрузка фото/видео не пойдёт с этим ключом — понадобится токен пользователя-администратора с правами photos/video (это отдельная настройка, подскажу при тесте).

## 7. `YOUTUBE_API_KEY` (для авто-сбора роликов YouTube)
- **https://console.cloud.google.com** → создай проект (кнопка вверху) → меню **APIs & Services → Library** → найди **YouTube Data API v3** → **Enable**.
- Затем **APIs & Services → Credentials → Create credentials → API key** → скопируй.
- Бесплатно в пределах суточной квоты.

## 8. `OK_APP_KEY` и `OK_APP_SECRET` (для автопостинга в OK; опционально)
- **https://ok.ru/devaccess** → подай заявку на доступ разработчика → создай приложение.
- В приложении получишь **Application ID / Public key** (`OK_APP_KEY`) и **Secret key** (`OK_APP_SECRET`).
- Для публикации нужен ещё access_token группы — подскажу при настройке. OK сложнее VK; можно начать без него.

## 9. `OPENAI_API_KEY` (для генератора обложек ИИ; опционально)
⚠️ Подписка **ChatGPT Plus НЕ подходит** — нужен отдельный API-ключ (оплата отдельно, по факту, картинка ~несколько центов).
- **https://platform.openai.com** → войди тем же аккаунтом ChatGPT.
- **Settings → Billing** → добавь карту и кредиты ($5–10 на старт).
- **API keys** (https://platform.openai.com/api-keys) → **Create new secret key** → скопируй.
- Опц. `OPENAI_IMAGE_MODEL` — по умолчанию `gpt-image-1`; при проблемах можно `dall-e-3`.

## 10. Telegram-разведка — ключ НЕ нужен
Публичные каналы собираются через `t.me/s/<канал>` без токена. Просто добавляешь ссылки на каналы.

---

## Итоговый список переменных в Railway (сервис ks-content)
Обязательные: `DATABASE_URL`, `SECRET_KEY`, `CONTENT_OWNER_EMAIL`, `CONTENT_OWNER_PASSWORD`, `ANTHROPIC_API_KEY`.
Для разведки: `VK_SERVICE_TOKEN`, `YOUTUBE_API_KEY`.
Для автопостинга OK: `OK_APP_KEY`, `OK_APP_SECRET`.
(VK-токен сообщества для постинга — вводится внутри платформы, не в Railway.)

Порядок: обязательные → подними сервис → войди владельцем → добавь остальные по мере надобности.
