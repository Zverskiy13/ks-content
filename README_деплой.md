# Контент-платформа регионов — деплой

Отдельное веб-приложение (FastAPI). Общая база Postgres с приложением и ботом, свои таблицы (`cp_*`). Регионы, вход по e-mail+паролю, агент-аналитик, контент-план, выбор площадок (VK/OK активны, IG/TikTok — на будущее).

## Куда деплоить
Новый **отдельный сервис** на Railway (не трогает приложение и бота), рядом, на том же Postgres.

1. Создай новый **репозиторий** на GitHub (напр. `ks-content`) и залей туда содержимое этой папки (`main.py`, `db.py`, `requirements.txt`, `Procfile`, `runtime.txt`, папка `web/`).
2. Railway → New → Deploy from GitHub repo → выбери `ks-content`. Создастся новый сервис.
3. На этом сервисе задай переменные (Variables):

| Name (печатать вручную) | Value |
|---|---|
| `DATABASE_URL` | публичная строка того же Postgres (Railway → проект keen-learning → Postgres → Connect → Public) |
| `SECRET_KEY` | любая длинная случайная строка (для подписи сессий) |
| `ANTHROPIC_API_KEY` | тот же ключ Claude, что у приложения |
| `CONTENT_OWNER_EMAIL` | твой e-mail для входа владельцем |
| `CONTENT_OWNER_PASSWORD` | пароль владельца (задай сам) |
| `VK_SERVICE_TOKEN` | (опц., авто-сбор VK) сервисный ключ доступа VK-приложения |
| `YOUTUBE_API_KEY` | (опц., авто-сбор YouTube) ключ YouTube Data API v3 из Google Cloud |
| `OK_APP_KEY`, `OK_APP_SECRET` | (опц., автопостинг в OK) публичный и секретный ключи OK-приложения |
| `OPENAI_API_KEY` | (опц., генератор обложек) ключ OpenAI API (platform.openai.com — НЕ подписка ChatGPT Plus) |
| `OPENAI_IMAGE_MODEL` | (опц.) модель картинок, по умолчанию `gpt-image-1`; можно `dall-e-3` |

### Как получить `VK_SERVICE_TOKEN` (для авто-сбора роликов)
1. Зайди на **vk.com/apps?act=manage** → **Создать приложение** → тип «Standalone» (или Embedded).
2. В приложении: раздел **Настройки** → скопируй **«Сервисный ключ доступа»**.
3. Вставь его в переменную `VK_SERVICE_TOKEN` контент-сервиса → Redeploy.

Без этого токена авто-сбор («Разведка») выключен, но всё остальное работает: ролики можно добавлять вручную во вкладке «Вирусное».

4. Deploy. Railway даст адрес вида `https://ks-content-xxxx.up.railway.app`.
5. Открой адрес в браузере на компьютере → войди под `CONTENT_OWNER_EMAIL` / `CONTENT_OWNER_PASSWORD`.

## Что внутри
- **Вирусное:** вставляешь ссылку/описание ролика → аналитик даёт карточку (тема, хук, структура, перенос механики на направления) + вердикт по ст.24 и метку «органика / реклама (нужен erid)».
- **Разведка (мультиплатформа):** добавляешь любые публичные аккаунты/каналы конкурентов из любой сети (смотреть чужой контент не запрещено). Авто-сбор роликов сейчас: **VK** (`VK_SERVICE_TOKEN`), **YouTube** (`YOUTUBE_API_KEY`) и **Telegram** (публичные каналы через t.me/s — ключ не нужен). Для **OK / Instagram / TikTok** авто-сбора нет (у OK нет открытого API чтения чужих групп; у IG/TikTok — тоже) — их источники хранятся, а конкретные ролики разбираешь по ссылке во «Вирусном». Ролики сортируются по просмотрам (общий сигнал для всех сетей).

### Как получить `YOUTUBE_API_KEY`
1. console.cloud.google.com → создай проект → **APIs & Services → Library** → включи **YouTube Data API v3**.
2. **Credentials → Create credentials → API key** → скопируй ключ → вставь в `YOUTUBE_API_KEY`.
- **Контент-план + автопостинг (v2.2):** собираешь пост, выбираешь площадки и дату/время. Подключаешь сообщества VK/OK (карточка «Подключение соцсетей»). Планировщик раз в минуту публикует записи со статусом **«Готово»**, у которых наступили дата/время — в подключённые VK/OK. Есть кнопка «Опубликовать» (сразу). Статус становится «Опубликовано» + ссылка, при ошибке — красный текст с причиной.
  - **VK:** нужен **токен сообщества** (Сообщество → Управление → Работа с API → Ключи доступа → создать с правами «Управление сообществом» и «Стена») + числовой **ID группы**. Публикует текст поста.
  - **OK:** нужны `OK_APP_KEY`/`OK_APP_SECRET` (OK-приложение) + **access_token группы** и её ID. OK-постинг реализован, но проверь после настройки приложения.
  - **Генератор обложек (ИИ, OpenAI):** в форме плана кнопка «Сгенерировать обложку» — ИИ рисует картинку по описанию (или по заголовку), она показывается превью и прикладывается к посту VK автоматически (передаётся напрямую, без хостинга). Нужен `OPENAI_API_KEY`. Промпт заворачивается в безопасный медицинский стиль (без «результатов лечения»).
  - **Картинка/видео во вложении (VK):** в форме плана поля «Ссылка на картинку» и «Ссылка на видеофайл (.mp4)» — платформа скачает по URL и приложит к посту VK. ⚠️ Для загрузки медиа токен VK должен иметь права **photos/video** (обычно токен пользователя-админа сообщества, а не всякий токен сообщества). Если не выйдет — пост уйдёт без медиа/текстом.
- **Результаты (контролёр):** вкладка «📊 Результаты» — статистика опубликованных постов (VK: просмотры/лайки/репосты/комменты), обновляется автоматически ~раз в 30 мин и кнопкой «Обновить статистику». Кнопка «Совет ИИ» даёт вывод: что масштабировать, что убрать, какие темы на следующий цикл.
- **Команда** (у владельца): добавляешь менеджеров регионов (видят только свой регион).

Регионы заводятся автоматически: Москва, Астрахань, Калмыкия, КБР, Чечня.

## Безопасность
- Пароли хранятся как scrypt-хеши; сессия — подписанная HttpOnly-cookie.
- Каждый менеджер видит только свой регион (проверка на сервере).
- Не задавай `SESSION_INSECURE` в проде (cookie только по HTTPS).
